Working with Dates and Times

There will be code and notes in this segment that deal with the “old” java date and time APIs as well as the new way introduced in Java 8. Even though we may mention the old way, we only have to focus on the.

Creating Dates and Times

There are four main types when dealing with dates and times, we will also have to be aware of time zones as well in OCJP.

LocalDate

Contains just a date—no time and no time zone. A good example of LocalDate is your birthday this year. It is your birthday for a full day, regardless of what time it is.

Looks like: 2015–05–25

LocalTime

Contains just a time—no date and no time zone. A good example of LocalTime is midnight. It is midnight at the same time every day.

Looks like: 09:13:07.768

LocalDateTime

Contains both a date and time but no time zone. A good example of LocalDateTime is “the stroke of midnight on New Year’s Eve.”

Looks like: 2015–05–25T09:13:07.768

ZonedDateTime

Contains a date, time, and time zone. A good example of ZonedDateTime is “a conference call at 9:00 a.m. EST.”

Looks like: 2015–05–25T09:13:07.769–04:00[America/New_York]

Time zones

Time zones are given via GMT or UTC, time zones can be denoted using +02:00, GMT+2, UTC+2, all have identical meaning.

You will also need to be able to calculate time zone differences.

2015–06–20T07:50–07:00[US/Pacific] 2015–06–20T07:50+05:30[Asia/Kolkata]

Here we can see the two regions are 12.5 hours apart, we could then use this difference to calculate the time in one region given the other.

eg What is it in [US/Pacific] when its 03:40 in [Asia/Kolkata]?

We can see that Asia is ahead of the US so we need to subtract 12.5 hours (12 hours and 30 minutes). Fortunately because clocks are on two, twelve hour cycles (am/pm) we knows its just half an hour difference on the other side of the am/pm cycle hence the time in the [US/Pacific] would be 3:10pm / 15:10 on the previous day.

All time zones and dates are given in the format of Year/Month/Day as standard, this in contrast to many english regions that give the date Day/Month/Year and burger-land which has the format Month/Day/Year.

To get any of the respective date time now we use the now() method.

LocalDate.now();
LocalTime.now();
LocalDateTime.now();
ZonedDateTime.now();

Creating a LocalDate

We can also use the of() method to construct a local date instance

LocalDate date1 = LocalDate.of(2015, Month.JANUARY, 20); // Month is an enum
LocalDate date2 = LocalDate.of(2015, 1, 20); // month is an int

Both use the (year/month/day) format.

Creating a LocalTime

When creating a time you can specify the hour, minute, second and nanosecond using the overloaded of() method.

LocalTime time1 = LocalTime.of(6, 15); // hour and minute
LocalTime time2 = LocalTime.of(6, 15, 30); // + seconds
LocalTime time3 = LocalTime.of(6, 15, 30, 200); // + nanoseconds

All of which take int as the type.

Creating a LocalDateTime

We essentially merge the above factory methods.

LocalDateTime dateTime1 = LocalDateTime.of(2015, Month.JANUARY, 20, 6, 15, 30); // Jan 20, 2015 at 6 hours, 15 minutes and 30 seconds

// or

LocalDate date1 = ...;
LocalTime time1 = ...;
LocalDateTime dateTime2 = LocalDateTime.of(date1, time1);

Creating a ZonedDateTime

Once we are essentially creating a local dat-time and appending a time zone to it, this can often lead to large factory methods such as

ZonedDateTime zoned1 = ZonedDateTime.of(2015, 1, 20, 6, 15, 30, 200, ZoneId.of("US/Eastern"));

There’s a lot of numbers as well making it a bit hard to read.

As such you can build a zoned dat-time using local time, local date or local date-time and the zone-id.

LocalDate date1 = ...;
LocalTime time1 = ...;
ZoneId zone = ...;
LocalDateTime dateTime1 = LocalDateTime.of(date1,time1);

ZonedDateTime zoned2 = ZonedDateTime.of(date1, time1, zone);
ZonedDateTime zoned3 = ZonedDateTime.of(dateTime1, zone);

Note, the only way to create any of the above instances is via the of() factory method, the classes above all have their constructors as private meaning you can NOT directly instantiate the class.

LocalDate date = new LocalDate(); //  Will NOT compile

The of() also validates input as well and cna return an exception.

LocalDate.of(2015, Month.JANUARY, 32) // throws DateTimeException

Manipulating Dates and Time

When working the Java 8 date and time classes it is important to remember that these classes are immutable, and, like strings have to have their new value re-assigned to a variable.

We can add and minus dates and time using each classes respective addDATEAMOUNT, minusDATEAMOUNT, addTIMEAMOUNT, minusTIMEAMOUNT

LocalDate date = LocalDate.of(2014, Month.JANUARY, 20);
date = date.plusDays(2);// 2014–01–22
date = date.plusWeeks(1); // 2014–01–29
date = date.plusMonths(1); // 2014–02–28
date = date.plusYears(5); // 2019–02–28

We can apply similar method calls to our local time class

LocalDate date = LocalDate.of(2020, Month.JANUARY, 20);
LocalTime time = LocalTime.of(5, 15);

LocalDateTime dateTime = LocalDateTime.of(date, time);  // 2020–01–20T05:15

dateTime = dateTime.minusDays(1); // 2020–01–19T05:15
dateTime = dateTime.minusHours(10); // 2020–01–18T19:15
dateTime = dateTime.minusSeconds(30); // 2020–01–18T19:14:30

Since each method returns a new instance of their respective class we can chain method calls together.

LocalDateTime dateTime = LocalDateTime.of(date, time).minusDays(1).minusHours(10).minusSeconds(30);

There are two main traps to avoid when dealing with dates and time.

  1. Like Strings LocalDate,LocalTime,LocalDateTime and ZonedDateTime are all immutable meaning unless the result is stored in a variable it is essentially lost.
LocalDate date = LocalDate.of(2020, Month.JANUARY, 20);
date.plusDays(10);
System.out.println(date); //2020-01-20

Here we did not assign the updated date to the date variable, so the result was never updated.

  1. Logically speaking we can only add date units to dates anf time units to time, hence it doesn’t make sense (nor will it compile) to add a time unit to a date or vice versa.
LocalDate date = LocalDate.of(2020, Month.JANUARY, 20);
date = date.plusMinutes(1); // DOES NOT COMPILE

Working with Periods.

While dates and time give us a snapshot or an instance in time, a Period gives us a duration or an extended notion of time.

Hence a date may be 2017-06-01 (2017 June 1) and another date maybe 2017-07-01 (2017 July 1). Here we have two dates, but the period between these dates is one month.

Period annually = Period.ofYears(1); // every 1 year
Period quarterly = Period.ofMonths(3); // every 3 months
Period everyThreeWeeks = Period.ofWeeks(3); // every 3 weeks
Period everyOtherDay = Period.ofDays(2); // every 2 days
Period everyYearAndAWeek = Period.of(1, 0, 7); // 1 year, 0 months and 7 days

Periods are static methods and can not be chained as such.

In the above you can see the helper methods such as ofYears(), ofMonths() etc. Under the hood these all passed to the of() method which takes a years, months and days.

When printing out a period, only non zero values will be printed appended by the value they represent, eg Y for year, M for Month and D for day. All periods are pre-fixed with a capital ‘P’.

System.out.printIn(Period.of(1,2,3));// P1Y2M3D

System.out.printIn(Period.of(0,0,3));// P3D

System.out.printIn(Period.ofWeeks(3));// P21D

System.out.println(Period.of(0, 20, 47));// P20M47D

The second last example prints 21 days as there are 21 days in three weeks.

The last example is also valid as twenty months is a valid period of time same as forty-seven days.

Essentially Periods only work on LocalDate and LocalDateTime objects.

LocalDate date = LocalDate.of(2015, 1, 20);
LocalTime time = LocalTime.of(6, 15);
LocalDateTime dateTime = LocalDateTime.of(date, time);

Period period = Period.ofMonths(1);

System.out.println(date.plus(period)); // 2015–02–20

System.out.println(dateTime.plus(period)); // 2015–02–20T06:15

System.out.println(time.plus(period)); // UnsupportedTemporalTypeException

Working with Durations

Durations are similar to Periods except they generally work on LocalTime andLocalDateTime and refer to hours,minutes and seconds instead of years, months and days. All duration values are pre-fixed with PT (Period of Time)

Duration daily = Duration.ofDays(1); // PT24H
Duration hourly = Duration.ofHours(1); // PT1H
Duration everyMinute = Duration.ofMinutes(1); // PT1M
Duration everyTenSeconds = Duration.ofSeconds(10); // PT10S
Duration everyMilli = Duration.ofMillis(1); // PT0.001S
Duration everyNano = Duration.ofNanos(1); // PT0.000000001S

You can also specify a scalar value and a ChronoUnit value to denote the type as well, using the of() method.

Duration daily = Duration.of(1, ChronoUnit.DAYS);
Duration hourly = Duration.of(1, ChronoUnit.HOURS);
Duration everyMinute = Duration.of(1, ChronoUnit.MINUTES);
Duration everyTenSeconds = Duration.of(10, ChronoUnit.SECONDS);
Duration everyMilli = Duration.of(1, ChronoUnit.MILLIS);
Duration everyNano = Duration.of(1, ChronoUnit.NANOS);

Like our example with period adding or subtracting a duration to a date will fail to compile

LocalDate date = LocalDate.of(2015, 1, 20);
LocalTime time = LocalTime.of(6, 15);
LocalDateTime dateTime = LocalDateTime.of(date, time);

Duration duration = Duration.ofHours(6);

System.out.println(dateTime.plus(duration)); // 2015–01–20T12:15 works because we have a time component.
System.out.println(time.plus(duration)); // 12:15

System.out.println(date.plus(duration)); // UnsupportedTemporalException

Working with Instants

An instant is a special ZonedDateTime which is ALWAYS in GMT time.

Instant now = Instant.now();
// do something time consuming
Instant later = Instant.now();

Duration duration = Duration.between(now, later);

System.out.println(duration.toMillis());

You can convert and ZonedDateTime object to an Instant by calling the toInstant() method

Instant instant = zonedDateTime.toInstant();

If the time zone in the zonedDateTime is in a different region to GMT, Java will automatically convert it to the GMT time zone.

An Instant also allows to do calculations, similar to the Duration that is it can calculate units that are a day or smaller. Using the ChronoUnit plus a scalar value.

Instant nextDay = instant.plus(1, ChronoUnit.DAYS);
System.out.println(nextDay); // 2015–05–26T15:55:00Z

Instant nextHour = instant.plus(1, ChronoUnit.HOURS);
System.out.println(nextHour); // 2015–05–25T16:55:00Z

Instant nextWeek = instant.plus(1, ChronoUnit.WEEKS); // exception

Accounting for Daylight Savings Time

The ZonedDateTime class understands that some regions have daylight savings time and recognises when to skip or rewind an hour based on the date and time zone.

LocalDate date = LocalDate.of(2016, Month.MARCH, 13);
LocalTime time = LocalTime.of(2, 30);
ZoneId zone = ZoneId.of("US/Eastern");
ZonedDateTime dateTime = ZonedDateTime.of(date, time, zone);
System.out.println(dateTime); // 2016–03–13T03:30–04:00[US/Eastern]

Here Java knows in the eastern parts of burger land that March 13, daylight savings changeover skips an hour forward and hence 2:30 am becomes 3:30am.

Reviewing the String class

When comparing String objects it is best to use the equals() method, to ensure we the values we are comparing are the same.

Referencing the same string literal from the String pool may also result in equality but this should not be relied upon.

String s1 = "bunny";
String s2 = "bunny";
String s3 = new String("bunny");

System.out.println(s1 == s2); // true
System.out.println(s1 == s3); // false
System.out.println(s1.equals(s3)); // true

We can use the + operator to concatenate strings together. If using the + operator where one operand is a string, Java will convert the other operand to a string.

Java evaluates expression from left to right.

String s4 = "1" + 2 + 3;
String s5 = 1 + 2 + "3";

System.out.println(s4); // 123
System.out.println(s5); // 33

The string class often uses 0-based indices in their methods

String s = "abcde "; //deliberate white space at the end
s.trim().length(); // 5
s.charAt(4); // e
s.indexOf('e'); // 4
s.indexOf("de"); // 3
s.substring(2, 4).toUpperCase(); // CD
s.replace('a', '1'); // 1bcde
s.contains("DE"); // false
s.startsWith("a"); // true

If we want to constantly update the value of a string, it is better to use a StringBuilder, if you require thread-safety you optionally use StringBuffer.

StringBuilder and StringBuffer are both mutable classes.

StringBuilder b = new StringBuilder();
b.append(12345).append('-');

b.length(); // 6
b.indexOf("-"); // 5
b.charAt(2); // 3

StringBuilder b2 = b.reverse();
b.toString(); // -54321
System.out.println(b == b2); // true

Since most of the builder classes methods return the builder instance we can often chain method calls.

StringBuilder s = new StringBuilder("abcde");

s.insert(1, '-').delete(3, 4); // a-bcde --> a-bde
System.out.println(s); //a-bde
System.out.println(s.substring(2, 4)); // bd

Adding Internationalization and Localization

Internationalisation refers to the ability of the program to adapt to other formats and languages. Localisation refers to being able to support a specific locale, where locale is defined as:

a specific geographical, political, or cultural region.

Picking a Locale

To get the users current locale we can use. Note java just refers the locale value from the underlying OS.

Locale locale = Locale.getDefault();
System.out.println(locale); // en_AU

A valid locale must have language element in lower-case, it can optionally a country as well, preceded by an underscore.

US // can have a language without a country, but not the reverse
enUS // missing underscore
US_en // the country and language are reversed
EN // language must be lowercase

Java allows three ways to create a Locale.

  1. You can use enums for common countries and languages, note denoting the language will just give you the language, while the name of the country will give you the country and the language
System.out.println(Locale.GERMAN); // de
System.out.println(Locale.GERMANY); // de_DE
  1. You can also use the constructor to create a new locale. When using the constructor it is important to care when entering the locale as java does not validate that the language or region is correct and lead to unexpected behaviours later on in the program. You also have to ensure correct casing when entering the locale.
System.out.println(new Locale("fr")); // fr
System.out.println(new Locale("hi", "IN")); // hi_IN
  1. Locale also comes with an in-built builder class, that can be used to build a locale, note unlike the constructor, the builder will correct your casing, but it still will NOT validate whether the locale actually exists.
Locale l2 = new Locale.Builder()
.setRegion("GB")
.setLanguage("en")
.build();

Locale l3 = new Locale.Builder() // bad but legal
.setRegion("au")
.setLanguage("EN")
.build();

You can also change the default locale using the setLocale() method.

Locale locale = new Locale("fr");
Locale.setDefault(locale); // change the default
System.out.println(Locale.getDefault()); // fr

This only changes the locale for that java program.

Using a Resource Bundle

A resource bundle contains the local specific objects to be used by a program. It is like a map with keys and values. The resource bundle can be in a property file or in a Java class. A property file is a file in a specific format with key/value pairs.

Generally speaking it is common to specify the resource bundle as an external file. This allows to update the localisation without having to restart or re-compile the program.

Creating a Property File Resource Bundle

To create valid resource bundle file we specify the resource name followed by and underscore, followed by the locale’s language.

Zoo_en.properties
  hello=Hello
  open=The zoo is open.

Zoo_fr.properties
  hello=Bonjour
  open=Le zoo est ouvert

To use the above files we can do the following:

import java.util.*;
public class ZooOpen {

public static void main(String[] args) {
  Locale us = new Locale("en", "US");
  Locale france = new Locale("fr", "FR");

  printProperties(us);
  System.out.println();
  printProperties(france);
}

public static void printProperties(Locale locale) {
  ResourceBundle rb = ResourceBundle.getBundle("Zoo", locale);
  System.out.println(rb.getString("hello"));
  System.out.println(rb.getString("open"));
 }
}

The resource bundle file can take key/value pairs in different ways 1. using the = sign key=value 2. using the : sign key:value 3. using a space key value

Java ignores any whitespace at the start of the line, but does NOT ignore white at the end of the line.

You can use ! or # to comment out a line.

You can retrieve all the keys from a resource bundle by using the keySet() method

Set<String> keys = rb.keySet();

keys.stream().map(k -> k + " " + rb.getString(k)).forEach(System.out::println);

A property file is similar to a resource bundle except it does NOT take a locale (just a file of key/value pairs).

To retrieve a value we can use the getProperty() method instead of the getString() method. The getProperty() also allows us to pass in a default value if the file has no value for that key.

System.out.println(props.getProperty("notReallyAProperty")); // returns null
System.out.println(props.getProperty("notReallyAProperty", "123")); // returns 123

Creating a Java Class Resource Bundle

We can also create a resource bundle object. This has the advantage that the value of the key doesn’t have to be a string.

It also has the advantage that we can manipulate properties at runtime, without having to read and write from a file.

To create an object resource bundle we must extend the ListResourceBundle abstract class.

import java.util.*;
public class Zoo_en extends ListResourceBundle {

  protected Object[][] getContents() {
    return new Object[][] {
      { "hello", "Hello" },
      { "open", "The zoo is open" } };
  } 
}

Here, Java looks at the class name delimited by the underscores. We can also specify the country as well by placing an underscore and stating the country.

package resourcebundles;
import java.util.*;

public class Tax_en_US extends ListResourceBundle {
  protected Object[][] getContents() {
    return new Object[][] { { "tax", new UsTaxCode() } };
  }

  public static void main(String[] args) {
    ResourceBundle rb = ResourceBundle.getBundle("resourcebundles.Tax", Locale.US);
    System.out.println(rb.getObject("tax"));
}}

Here we use the getObject() method to retrieve an object instead of string.

To use the bundle we specify the name of the package and class, relative to our classpath. Hence our bundle class is called Tax and it is in the resroucebundles package.

Determining Which Resource Bundle to Use

If no locale is specified when requesting a resource bundle, Java will assume the system default locale.

When looking for resource bundles Java, will look for the most specific match, language + country, then language, then it will assume the default locale. Java first looks for class files then properties files.

Java’s resource bundles are hierarchical (Actually it’s more similar to JS prototype chain).

Assuming there are valid property files or classes; Zoo_fr_FR.java, Zoo_fr.java, Zoo.java.

If the requested resource bundle is for Zoo_fr_FR.java than java can retrieve keys from any other of classes.

If we have all the same classes again, but the requested resource bundle was Zoo_fr.java.

Then Java could retrieve keys from Zoo_fr.java, Zoo.java.

Hence we could have the following property files.

Zoo.properties
  name=Vancouver Zoo

Zoo_en.properties
  hello=Hello
  open=is open

Zoo_en_CA.properties
  visitor=Canada visitor

Zoo_fr.properties
  hello=Bonjour
  open=est ouvert

Zoo_fr_CA.properties
  visitor=Canada visiteur

And the following code:

Locale locale = new Locale("en", "CA");
ResourceBundle rb = ResourceBundle.getBundle("Zoo", locale);

System.out.print(rb.getString("hello"));
System.out.print(". ");

System.out.print(rb.getString("name"));
System.out.print(" ");

System.out.print(rb.getString("open"));
System.out.print(" ");

System.out.print(rb.getString("visitor"));

This prints out: “Hello. Vancouver Zoo is open Canada visitor.”

Here because we specified both the language and the country we can also access the more generic files as well.

Formatting Numbers

We can format numbers using the NumberFormat class and any valid Locale.

Format and Parse Numbers and Currency

The format class can be used to format number, currency,percentages or decimals;

Number:

NumberFormat.getInstance();
NumberFormat.getInstance(locale);

Currency:

NumberFormat.getCurrencyInstance();
NumberFormat.getCurrencyInstance(locale);

Percent:

NumberFormat.getPercentInstance();
NumberFormat.getPercentInstance(locale);

Formatting

We can use the format() method to turn the number into a formatted string or you can call parse() to turn a string into a number.

import java.text.*;
import java.util.*;

public class FormatNumbers {
  public static void main(String[] args) {
    int attendeesPerYear = 3_200_000;
    int attendeesPerMonth = attendeesPerYear / 12;

    NumberFormat us = NumberFormat.getInstance(Locale.US);
    System.out.println(us.format(attendeesPerMonth));

    NumberFormat g = NumberFormat.getInstance(Locale.GERMANY);
    System.out.println(g.format(attendeesPerMonth));

    NumberFormat ca = NumberFormat.getInstance(Locale.CANADA_FRENCH);
    System.out.println(ca.format(attendeesPerMonth));
  } 
}

Gives the following output:
266,666 US
266.666 GER
266 666 CANFR

When we call the respective getxxxInstance() without a locale, Java will use the system default locale.

double price = 48;
NumberFormat aud = NumberFormat.getCurrencyInstance();
System.out.println(aud.format(price)); // $48.00

Parsing

Parsing goes the other way from formatting, when we parse, we parse an already formatted string or number and return the number.

Java parses the input based on the locale as well, and will return the result when the whole input is parsed.

If Java encounters a character that it can not parse for that particular locale Java will stop parsing and return what it’s understood. If no characters can be parsed Java throws a ParseException.

NumberFormat en = NumberFormat.getInstance(Locale.US);
NumberFormat fr = NumberFormat.getInstance(Locale.FRANCE);

String s = "40.45";

System.out.println(en.parse(s)); // 40.45
System.out.println(fr.parse(s)); // 40

We can call the parse method on currency strings as well to get the number.

String amt = "$92,807.99";
NumberFormat cf = NumberFormat.getCurrencyInstance();
double value = (Double) cf.parse(amt); // parse() returns a Number object, hence we have to cast to a double.
System.out.println(value); // 92807.99

Formatting Dates and Time

Java 8 provides the DatTimeFormatter class to help with formatting local-time, local-date, local-date-time and zoned-date-time objects.

The DateTimeFormatter can be used to format any date or time object even the pre Java 8 date and time classes.

LocalDate date = LocalDate.of(2020, Month.JANUARY, 20);
LocalTime time = LocalTime.of(11, 12, 34);
LocalDateTime dateTime = LocalDateTime.of(date, time);

System.out.println(date.format(DateTimeFormatter.ISO_LOCAL_DATE)); // 2020–01–20
System.out.println(time.format(DateTimeFormatter.ISO_LOCAL_TIME)); // 11:12:34
System.out.println(dateTime.format(DateTimeFormatter.ISO_LOCAL_DATE_TIME)); // 2020–01–20T11:12:34

DateTimeFormatter also has a ofLocalizedDate() method which can return the formatted date/time in the users locale. There are also pre-defined styles that are common and can be accessed via FormatStyle.SHORT or FormatStyle.MEDIUM

DateTimeFormatter shortDateTime =
DateTimeFormatter.ofLocalizedDate(FormatStyle.SHORT);

System.out.println(shortDateTime.format(dateTime)); // 20/01/20
System.out.println(shortDateTime.format(date)); // 20/1/20

System.out.println(shortDateTime.format(time)); // UnsupportedTemporalTypeException as the ofLocalizedDate method expects some part of the string to be a date.

For formatting time and date-time objects we can use the respective

ofLocalizedDateTime() for dates and times as well as date-time

ofLocalizedTime() ONLY for times

ofLocalizedDate() ONLY for dates

LocalDate date = LocalDate.of(2020, Month.JANUARY, 20);
LocalTime time = LocalTime.of(11, 12, 34);
LocalDateTime dateTime = LocalDateTime.of(date, time);

DateTimeFormatter shortF = DateTimeFormatter.ofLocalizedDateTime(FormatStyle.SHORT);
DateTimeFormatter mediumF = DateTimeFormatter.ofLocalizedDateTime(FormatStyle.MEDIUM);

System.out.println(shortF.format(dateTime)); // 1/20/20 11:12 AM
System.out.println(mediumF.format(dateTime)); // Jan 20, 2020 11:12:34 AM

We can also make custom formats using the ofPattern() method.

DateTimeFormatter f = DateTimeFormatter.ofPattern("MMMM dd, yyyy, hh:mm");
System.out.println(dateTime.format(f)); // January 20, 2020, 11:12

MMMM, M represents the month. The more Ms you have, the more verbose the Java output. For example, M outputs 1, MM outputs 01, MMM outputs Jan, and MMMM outputs January.

dd, d represents the day in the month. As with month, the more ds you have, the more verbose the Java output. dd means to include the leading zero for a single-digit day.

, Use , if you want to output a comma (this also appears after the year).

yyyy y represents the year. yy outputs a two-digit year and yyyy outputs a four-digit year.

hh, h represents the hour. Use hh to include the leading zero if you’re outputting a single-digit hour.

:, Use : if you want to output a colon.

mm, m represents the minute omitting the leading zero if present. mm is more common and represents the minutes using two digits.

Tip du jour: Incorpriating bootstrap styling in angular 2/4 with ng-cli

 Jun 6, 2017

You can add any css files to the angular-cli.json styles field to have them globally added to your angular 2⁄4 project

"styles": [
         "../node_modules/bootstrap/dist/css/bootstrap.min.css",
        "styles.css",
      ]

Here we added boostrap to our global styles, so now we can do the following without any extra imports.

<div class="container">
  <div class="panel panel-primary">
  {{componentHeader}}
  </div>  
</div>
